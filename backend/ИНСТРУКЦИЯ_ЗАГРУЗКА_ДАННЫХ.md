# Инструкция по загрузке данных в таблицу ready_sales

## Описание
Данная инструкция описывает процесс загрузки данных из Excel файлов (ГОТОВЫЙ.xlsx) в таблицу `ready_sales` базы данных.

## Структура таблицы ready_sales

Таблица содержит следующие поля:
- `data` - Дата (DateField)
- `diler` - Дилер
- `klient_id` - ID клиента
- `klient` - Название клиента
- `invoice_sid` - ID инвойса
- `tip` - Тип
- `tip_organizacii` - Тип организации
- `produkt` - Продукт
- `gruppa_produktov` - Группа продуктов
- `kolichestvo` - Количество
- `ves_kg` - Вес в кг
- `obshchaya_summa` - Общая сумма
- `dokhod` - Доход
- `valyuta` - Валюта
- `tovary` - Товары
- `year` - Год (для разделения данных по годам)
- `sheet_name` - Название листа Excel (Лист1, Лист2, Лист3, Лист4)

## Процесс загрузки данных (2 этапа)

### ⚠️ ВАЖНО
Загрузка происходит в **ДВА ЭТАПА** для оптимизации:
1. **Экспорт Excel → JSON** (делается один раз, медленно)
2. **Импорт JSON → БД** (можно повторять, быстро)

---

## Этап 1: Экспорт Excel → JSON

### Подготовка
1. Поместите Excel файлы в папку `backend/`:
   - `2024 ГОТОВЫЙ.xlsx`
   - `2025 ГОТОВЫЙ.xlsx`
   - `2026 ГОТОВЫЙ.xlsx` (когда появится)
   - и т.д.

### Структура Excel файлов
- Каждый файл должен содержать **4 листа**: Лист1, Лист2, Лист3, Лист4
- Каждый лист должен иметь стандартные колонки:
  - Дата
  - Дилер
  - Клиент ИД
  - Клиент
  - Инвоиcе CИД
  - Тип
  - Тип организации
  - Продукт
  - Группа продуктов
  - Количество
  - Вес(кг)
  - Общая сумма
  - Доход
  - Валюта
  - ТОВАРЫ

### Команда для экспорта
```powershell
cd d:\2026\sales-analytics\backend
python export_excel_to_json.py
```

### Что делает скрипт
- Сканирует все файлы вида `XXXX ГОТОВЫЙ.xlsx`
- Извлекает год из названия файла (2024, 2025, 2026 и т.д.)
- Читает **ВСЕ 4 листа** из каждого файла
- Создает отдельный JSON файл для каждого листа:
  - `2024_Лист1.json`
  - `2024_Лист2.json`
  - `2024_Лист3.json`
  - `2024_Лист4.json`
  - `2025_Лист1.json`
  - и т.д.

### Время выполнения
- ~3-5 минут на один Excel файл (зависит от размера)
- JSON файлы занимают ~2.8 ГБ для 2024-2025 годов

## Этап 2: Импорт JSON → БД

### Команда для импорта
```powershell
cd d:\2026\sales-analytics\backend
python load_from_json.py
```

### Что делает скрипт
1. Находит все JSON файлы вида `202*_*.json`
2. Спрашивает, нужно ли удалить существующие данные
   - **yes/y/да/д** - удалит старые данные (полная перезагрузка)
   - **no/n/нет/н** - добавит к существующим данным
3. Загружает данные партиями (batch_size = 5000)
4. Выводит статистику по годам

### Время выполнения
- ~10-20 минут на все файлы
- Скорость: ~5000-10000 записей/секунду

## Добавление новых годов

### Когда появятся файлы 2026, 2027 и т.д.:

1. **Положите новый файл** `XXXX ГОТОВЫЙ.xlsx` в папку `backend/`

2. **Экспортируйте в JSON** (скрипт автоматически найдет новый файл):
   ```powershell
   python export_excel_to_json.py
   ```

3. **Загрузите в БД**:
   ```powershell
   python load_from_json.py
   ```
   - Выберите **no** при вопросе об удалении данных
   - Новые данные добавятся к существующим

### Важно
- Все данные хранятся в **ОДНОЙ** таблице `ready_sales`
- Разделение по годам происходит через поле `year`
- Скрипты автоматически определяют год из названия файла

## Скрипты

### 1. `export_excel_to_json.py`
Экспортирует ВСЕ листы из Excel файлов в JSON

### 2. `load_from_json.py`
Загружает JSON файлы в таблицу ready_sales

### 3. `check_excel_sheets.py`
Проверяет структуру Excel файлов (количество листов)

### 4. `analyze_ready_files.py`
Анализирует структуру Excel файлов (колонки, типы данных)

## Типичные ошибки и решения

### Ошибка: "Файл не найден"
**Решение:** Убедитесь, что Excel файл находится в папке `backend/`

### Ошибка: "ModuleNotFoundError"
**Решение:** Установите зависимости:
```powershell
pip install pandas openpyxl
```

### Экспорт занимает слишком много времени
**Решение:** Это нормально, файлы большие. Просто дождитесь завершения.

### База данных переполнена
**Решение:** 
1. Удалите старые данные: `ReadySale.objects.all().delete()`
2. Или используйте фильтры по годам
