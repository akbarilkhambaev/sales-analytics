# ТЗ: Мультисекторная система

**Дата:** 07.05.2026  
**Ветка разработки:** `feature/multi-sector`  
**Статус:** Планирование

---

## Цель

Разделить все операционные данные (расходы, задачи, отчёты, KPI, клиенты) по организационным подразделениям (секторам), без каких-либо изменений на фронтенде. Фронтенд автоматически получает уже отфильтрованные данные в зависимости от сектора вошедшего пользователя.

---

## Ключевые правила

- Менеджер принадлежит **только одному** сектору → видит только свои данные
- ADMIN / SUPER_ADMIN → `sector = null` → видит **все** секторы
- Секторы **динамические** — создаются через API/админку, без хардкода
- Существующие данные переносятся в один дефолтный сектор «Основной»
- Фронтенд **не меняется**

---

## Что НЕ меняется (глобальные данные)

| Таблица | Причина |
|---|---|
| `Sale`, `ReadySale`, `SalesPlan` | Аналитика продаж — глобальная |
| `TovaryMapping`, `SchetaMapping`, `DilerMapping` | Общие справочники |
| `KPITemplate`, `KPITemplateItem` | Общие шаблоны KPI |
| `ProductCatalog` | Общий каталог |
| Весь фронтенд | Без изменений |

## Бэкенд — Django

### Шаг 1. Новая модель `Sector` (`api/models.py`)

```python
class Sector(models.Model):
    name = models.CharField(max_length=100, unique=True, verbose_name='Название')
    code = models.SlugField(max_length=50, unique=True, verbose_name='Код (slug)')
    description = models.TextField(blank=True, verbose_name='Описание')
    is_active = models.BooleanField(default=True, verbose_name='Активен')
    created_at = models.DateTimeField(auto_now_add=True)
    created_by = models.ForeignKey(
        'auth.User', null=True, blank=True,
        on_delete=models.SET_NULL,
        related_name='sectors_created'
    )

    class Meta:
        db_table = 'sectors'
        verbose_name = 'Сектор'
        verbose_name_plural = 'Секторы'
        ordering = ['name']

    def __str__(self):
        return self.name
```

#### Data migration — дефолтный сектор

Создать файл `api/migrations/XXXX_data_migrate_default_sector.py`:

```python
def forwards(apps, schema_editor):
    Sector = apps.get_model('api', 'Sector')
    Expense = apps.get_model('api', 'Expense')
    ExpenseCategory = apps.get_model('api', 'ExpenseCategory')
    WorkReport = apps.get_model('api', 'WorkReport')
    KanbanColumn = apps.get_model('api', 'KanbanColumn')
    ClientCard = apps.get_model('api', 'ClientCard')
    KPIRecord = apps.get_model('api', 'KPIRecord')

    default = Sector.objects.create(name='Основной', code='main')

    for Model in [Expense, ExpenseCategory, WorkReport, KanbanColumn, ClientCard, KPIRecord]:
        Model.objects.filter(sector__isnull=True).update(sector=default)
```

### Шаг 5. Базовый mixin фильтрации (`api/utils.py`)

```python
class SectorQuerysetMixin:
    """
    Автоматически фильтрует queryset по сектору пользователя.
    ADMIN/SUPER_ADMIN (sector=None) видят все данные без фильтра.
    """
    sector_field = 'sector'  # переопределить если путь другой

    def get_queryset(self):
        qs = super().get_queryset()
        user = self.request.user
        profile = getattr(user, 'profile', None)

        if profile and profile.sector_id is not None:
            qs = qs.filter(**{self.sector_field: profile.sector_id})
        # ADMIN/SUPER_ADMIN (sector=None) — без фильтра
        return qs
```

### Сценарии для проверки

| # | Действие | Ожидаемый результат |
|---|---|---|
| 1 | Создать 2 сектора: «Сектор А», «Сектор Б» | Оба в БД, `is_active=True` |
| 2 | Создать менеджера в «Секторе А» | `UserProfile.sector_id = <id Сектора А>` |
| 3 | Создать расход в «Секторе А» | `Expense.sector = Сектор А` |
| 4 | Логин менеджера Сектора А → `GET /api/expenses/` | Видит **только** расходы Сектора А |
| 5 | Логин менеджера Сектора Б → `GET /api/expenses/` | Видит **только** расходы Сектора Б |
| 6 | Логин ADMIN (sector=null) → `GET /api/expenses/` | Видит **все** расходы обоих секторов |
| 7 | Менеджер А создаёт Kanban-колонку | Колонка привязана к Сектору А |
| 8 | Менеджер Б → `GET /api/kanban/columns/` | Видит только колонки Сектора Б |
| 9 | Менеджер А создаёт задачу в своей колонке | Задача доступна только Сектору А |
| 10 | `GET /api/sectors/` от менеджера | Возвращает список активных секторов |
| 11 | `DELETE /api/sectors/{id}/` от менеджера | 403 Forbidden |
| 12 | `DELETE /api/sectors/{id}/` от ADMIN | `is_active=False`, запись в БД остаётся |
| 13 | Существующие данные (data migration) | Все старые записи → `sector = 'Основной'` |
